スキル アーキテクチャ
スキルは、Claude がファイルシステム アクセス、bash コマンド、コード実行機能を持つコード実行環境で実行されます。これは次のように考えてください。スキルは仮想マシン上のディレクトリとして存在し、Claude はコンピューター上のファイルをナビゲートするために使用するのと同じ bash コマンドを使用してそれらと対話します。
エージェント スキル アーキテクチャ - スキルがエージェントの構成と仮想マシンにどのように統合されるかを示す

Claude がスキル コンテンツにアクセスする方法：
スキルがトリガーされると、Claude は bash を使用してファイルシステムから SKILL.md を読み込み、その指示をコンテキスト ウィンドウに取り込みます。これらの指示が他のファイル（FORMS.md やデータベース スキーマなど）を参照している場合、Claude は追加の bash コマンドを使用してそれらのファイルも読み込みます。指示が実行可能スクリプトについて言及している場合、Claude は bash 経由でそれらを実行し、出力のみを受け取ります（スクリプト コード自体はコンテキストに入りません）。
このアーキテクチャが実現するもの：
オンデマンド ファイル アクセス：Claude は各特定のタスクに必要なファイルのみを読み込みます。スキルには数十の参照ファイルを含めることができますが、タスクが販売スキーマのみを必要とする場合、Claude はそのファイルのみを読み込みます。残りはファイルシステムに残り、ゼロ トークンを消費します。
効率的なスクリプト実行：Claude が validate_form.py を実行すると、スクリプトのコードはコンテキスト ウィンドウに読み込まれません。スクリプトの出力のみ（「検証に成功しました」や特定のエラー メッセージなど）がトークンを消費します。これにより、スクリプトは Claude が同等のコードをその場で生成するよりもはるかに効率的になります。
バンドル コンテンツに実質的な制限なし：ファイルはアクセスされるまでコンテキストを消費しないため、スキルは包括的な API ドキュメンテーション、大規模なデータセット、広範な例、または必要な参照資料を含めることができます。使用されないバンドル コンテンツに対するコンテキスト ペナルティはありません。
このファイルシステムベースのモデルが段階的な情報開示を機能させるものです。Claude はオンボーディング ガイドの特定のセクションを参照するのと同じように、スキルをナビゲートし、各タスクが必要とするものに正確にアクセスします。

Agent Skillsの仕組みを支える「段階的開示」という考え方
Agent Skillsの核心には、「Progressive Disclosure（段階的開示）」という非常に賢い設計思想があります。これは、情報を一度にすべて見せるのではなく、必要な情報だけを、必要なタイミングで段階的にAIに開示することで、コンテキストウィンドウを効率的に利用し、AIの「迷い」を最小限に抑える仕組みです。

この段階的開示は、大きく3つのステップで構成されています。

ステップ1: メタデータで必要なスキルを瞬時に判断
AIエージェントは起動時に、インストールされている全てのスキルについて、その本体（詳細な指示書）を読み込むことはしません。代わりに、各スキルの**「名前（name）」と「説明（description）」というメタデータだけ**をシステムプロンプトに読み込みます。

これは、分厚いマニュアルの「目次」だけを最初に読むようなものです。AIはまず、どんなスキル（道具箱）が利用可能かを大まかに把握します。この初期情報が非常に少ないため、大量のスキルがインストールされていても、コンテキストウィンドウを圧迫せず、思考の邪魔になりません。

ステップ2: 関連するスキル本体（SKILL.md）を読み込む
ユーザーからタスクの指示を受け、AIが「このタスクには、あのスキルが役立ちそうだ」と判断したとします。例えば、「このPDFの申請書に記入して」という指示に対して、「PDFスキル」が関連すると判断した場合です。

このタイミングで初めて、AIはPDFスキルの本体であるSKILL.mdというファイルを読み込みます。このファイルには、タスクを遂行するためのより詳細な手順やルールが記述されています。マニュアルの目次を見て、該当する章を初めて開くイメージです。これにより、AIは無関係な情報に惑わされることなく、目の前のタスクに必要な知識だけを得て、思考を深めることができます。

ステップ3: 追加ファイルやコードでより複雑なタスクを実行
スキルによっては、非常に複雑な手順や、特定の状況でしか使わない情報、あるいはプログラムコードが必要になる場合があります。これら全てをSKILL.mdに記述すると、ファイルが長大になりすぎてしまいます。

そこでAgent Skillsでは、さらに詳細な情報を別のファイルに分割し、SKILL.mdから参照する仕組みが用意されています。例えば、「PDFスキル」の中に、フォーム入力に特化した指示書forms.mdや、参考情報reference.md、さらにはPDFからフォーム情報を自動抽出するPythonスクリプトextract_forms.pyなどを同梱しておくことができます。

AIはSKILL.mdを読んだ後、タスクの状況に応じて「フォーム入力が必要だから、forms.mdを読んでみよう」「この処理はPythonスクリプトを実行した方が早そうだ」と判断し、必要なファイルだけを追加で読み込んだり、スクリプトを実行したりします。これにより、コンテキストウィンドウの消費を最小限に抑えながら、極めて高度で専門的なタスクを遂行できるのです。
